## Risks / standing decisions

- **arm64 host vs amd64 pin — RESOLVED 2026-07-22 by the native-first
  pivot.** The original decision (build under `--platform linux/amd64` /
  Rosetta for CI comparability) hit exactly the predicted "unworkable"
  branch: GNU Make's jobserver breaks under emulation (forcing `-j1`)
  and serial emulated compiles are prohibitively slow. Per the DESIGN.md
  §9 revision, dev builds are now native arm64 on the host; a pinned
  Linux container is the canonical builder from M3 onward (arm64 per the
  same-day §9 amendment — the amd64 requirement is dropped), where a
  three-way artifact-hash equivalence check closes the
  host-arch-independence question properly.
- **ISO URL rotation.** Yearly CTAN URLs die when the next TL ships; only
  historic-archive mirrors are pin-safe. fetch.sh must never encode a
  "current" URL.
- **Upstream has no container.** The Dockerfile is original work (ours,
  MIT); only the build machinery it invokes is vendored busytex.
- **Provenance discipline.** Research around WASM TeX inevitably surfaces
  GPL wrappers; per DESIGN.md §2 their sources are never opened. Note any
  such encounter in the commit message of the work that avoided it.
