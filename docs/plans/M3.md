## Risks / standing decisions

- **No container builds on the local machine** (user directive
  2026-07-23, issued mid-item-5): neither the arm64 Linux nor the
  amd64 toolchain container runs here again. The two completed local
  container builds (item 4's and item 5's build #2) are the standing
  evidence base; every subsequent container build — item 6's amd64
  lane, any repro re-run, item 8's acceptance rebuild — happens on CI
  runners (item 7's infrastructure). Native macOS builds
  (`make artifacts`) remain unrestricted. This re-sequences the tail
  of the milestone: item 7's CI build capability becomes the
  prerequisite for finishing item 6's amd64 lane, so expect item 7
  (or its build-workflow slice) to land before item 6 closes.

- **Runner disk (14 GB)** is the hard wall: the full ISO extraction
  (6.5 GB) + source tree + work tree cannot coexist. Selective
  staging is load-bearing, not an optimization; if the basic-scheme
  package list proves fragile to enumerate, fall back to caching a
  pre-staged texmfrepo tarball produced by a scheduled full job.
- **Container build wall-time** on 4-vCPU arm64 runners: local M-series
  took ~70 min; expect 2-3× in CI. Within the 6 h job cap, but
  caching must make it a rare event (main/schedule, not per-PR).
- **Nondeterminism discoveries** at item 5 are findings, not
  surprises: gzip mtime in .fmt (gzip -n equivalent?), archive
  member ordering, and locale sorting are the classic suspects; fix
  at the source, never by fuzzy comparison.
- **The macOS lane** of the equivalence check uses the DEV toolchain
  (documented-not-pinned host tools) — hash differences attributable
  to the host layer (vs the container pair) are expected findings,
  not failures; the container pair is the canonical comparison.
- **emsdk linux-arm64 at 3.1.43**: assumed available (emsdk published
  arm64 Linux binaries in that era); if the prebuilt is missing, the
  decision rule mirrors M2's: minimum exact bump, pinned everywhere,
  journaled — and the native path re-verifies against it.
