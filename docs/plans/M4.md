# M4 ‚Äî Bundles + manifests: execution plan

Status: **COMPLETE 2026-07-24** ‚Äî all 9 items done, independently
accepted (item 9) against the CI-built tiered artifact. Next is M5
(release engineering) ‚Äî user-gated, since it includes the first real
asset release + npm publish. Below is the plan as executed.

M4 turns the single
`texlive-basic` data bundle into **tlpdb-driven tiers** (`core` /
`academic` / `full`), emits a **top-level integrity manifest** with a
per-bundle provided-package index, and implements the ¬ß5.4 **on-demand
resolution** the runtime API already forward-declares (typed but inert).
This is the milestone that makes the `bundles: { preload, onDemand }`
config real. Grounded in the 2026-07-24 codebase map (docs/LOG.md).

## What already exists (leverage, don't rebuild)

- **One bundle:** `texlive-basic.{js,data}` = install-tl `scheme-basic`
  + `collection-xetex` + `collection-latex`, file_packager'd (LZ4,
  52.7 MB). The `build/texlive-%` Makefile rule is generic ‚Äî only ever
  run with `%=basic`.
- **tlpdb on disk, read by nothing:** `source/texmfrepo/tlpkg/texlive.tlpdb`
  (ISO-staged). The exact `package ‚Üí files` map lives here; nothing
  parses it. Current provided-package signal is a crude
  `\ProvidesPackage` grep dumped as dead JS comments atop the bundle
  `.js` (`build/engines/Makefile:601`).
- **`assets.json` schemaVersion 1** (`build/manifest/gen-assets.mjs`):
  full per-file `{path,bytes,sha256,role}` inventory, deterministic.
  Its header + README explicitly call it the M4 manifest precursor.
- **`build/bundles/`** exists as a README-only stub.
- **Runtime `onDemand` plumbing** is typed + validated + copied but
  never consumed; `stats.bundlesLoaded` just echoes `preload`.
  Diagnostics carry the missing filename verbatim in `message` (a
  contract `diagnostics.ts` promises M4) but not as a structured field.
- **Corpus** (4 entries) stays entirely inside basic ‚Äî no test yet
  forces a second tier.

## Design decisions (this plan commits to these; open ones flagged)

- **Tiers are DISJOINT file sets.** Every TL file lands in exactly one
  of core/academic (and any future tier), so on-demand bundles mount on
  top of core without file collisions. Assignment is by tlpdb
  collection ‚Üí tier.
  - **`core`** = today's basic content (scheme-basic + xetex + latex
    collections) + the retained `.fmt`s. The always-preloaded tier;
    `texlive-basic` is renamed to `core` (back-comptible alias kept one
    release ‚Äî see risks).
  - **`academic`** = the **scientific-journal + CJK working set**
    (user target, 2026-07-24). TL collections:
    `collection-latexrecommended` (babel, natbib, psnfss),
    `collection-mathscience` (mathtools, siunitx, physics, unicode-math
    ‚Äî the journal math stack), `collection-latexextra` (geometry,
    hyperref, booktabs, subcaption, microtype, algorithms, listings,
    AND the journal classes: IEEEtran, elsarticle, revtex4, achemso‚Ä¶),
    `collection-pictures` (tikz/pgf/pgfplots ‚Äî scientific figures),
    `collection-fontsrecommended` (Latin Modern, TeX Gyre),
    `collection-langchinese` (ctex, xeCJK, zhnumber, ctexart/book ‚Äî the
    XeTeX Chinese path), and `collection-langcjk` (general CJK package
    for JP/KR in mixed docs). **Plus the `fandol` font** (~10 MB) as a
    deliberate exception to ¬ß6.3's "no bundled fonts": it is ctex's
    default free Chinese font, so a Chinese doc COMPILES out of the box
    rather than erroring until the host wires `\setCJKmainfont`. Heavier
    production CJK fonts (Noto CJK, Source Han) stay host-supplied per
    ¬ß6.3. Record as a ¬ß6.3 refinement at item 3.
  - **`full`** = **DEFERRED / not shipped for v1** (decided 2026-07-24
    against the journal+CJK target). TeX Live's long tail (exotic
    languages, humanities, presentation themes) is outside the target;
    `academic` IS the working set. The ¬ß5.4(b) missing-file retry gives
    a clean "package not in any loaded bundle" failure for outliers,
    which we add to `academic` on request ‚Äî cheaper than shipping/‚Äã
    building a multi-GB tier nobody in-target loads. Keeps CI builds
    fast (matches texlyre-busytex's real-world ~400 MB, no-full posture,
    observed independently ‚Äî no code derived). Revisit if a
    general-purpose (non-journal) audience emerges. The bundle
    machinery (items 2‚Äì5) stays N-tier-general so adding `full` later
    is config, not rework.
- **Manifest:** evolve `assets.json` ‚Üí **`manifest.json`
  (schemaVersion 2)**, superset-compatible: keep the per-file
  inventory, ADD `texliveSnapshot` (the TL 2026 freeze id + tlpdb
  revision), `engines` (named list), and `bundles: [{ name, files[],
  bytes, provides: [package‚Ä¶] }]`. The runtime consumes this; hosts
  verify installs against it (DESIGN ¬ß7). Keep `assets.json` as a
  generated alias for one release for back-compat.
- **On-demand mount:** load a tier's file_packager `.data` into MEMFS
  **after** engine init (not only preRun). **Open technical risk:**
  confirm Emscripten's `runWithFS`/`installDataPackage` works
  post-init in our worker; if the packaged loader assumes preRun only,
  fall back to WORKERFS or a re-init-with-more-preloads path. Item 5
  resolves this with a spike before the full runtime wiring.
- **¬ß5.4 resolution, both paths:** (a) static `\usepackage` /
  `\RequirePackage` scan of project sources (skip project-local
  `.sty`/`.cls`) ‚Üí map names via the manifest `provides` index ‚Üí ensure
  those bundles; (b) on a pass failing with kpathsea `File \`x' not
  found` for a name the manifest maps to an unloaded on-demand bundle
  ‚Üí load it, retry once. No network anywhere.

## Acceptance (DESIGN ¬ß5.4 + ¬ß7 + ¬ß9 M4 bullet)

- The build produces **2 disjoint tier bundles** ‚Äî `core` (‚âà today's
  basic) and `academic` (the journal+CJK working set above, strictly
  additive) ‚Äî each a file_packager `.js`+`.data`, from the pinned
  tlpdb. The generator stays **N-tier-general** so a `full` tier is
  later config, not rework. A Chinese doc using `ctex` compiles against
  core+academic out of the box (fandol bundled).
- **`manifest.json` (schemaVersion 2)** carries the TL snapshot id, the
  engine list, and a per-bundle `provides` index derived from the
  tlpdb; every file has `{bytes, sha256}`; it verifies against
  `SHA256SUMS`; deterministic under `SOURCE_DATE_EPOCH`.
- The runtime **loads on-demand bundles for real**: `bundlesLoaded`
  reflects actually-mounted tiers; `preload: ['core']` +
  `onDemand: ['academic']` works cold and storage-less.
- **¬ß5.4 both paths pass tests:** a corpus/fixture doc that
  `\usepackage`es something only in `academic` typesets green via (a)
  the static scan, and another via (b) the missing-file retry (scan
  disabled) ‚Äî proving the retry mechanism independently.
- **Citation is a first-class tested path:** a journal-style paper with
  `natbib` + `bibtex8` (real `.bst`) resolves `\citep`/`\citet` to a
  correct bibliography via the multi-pass sequence. (biber remains out
  of scope ‚Äî see risks.)
- **CI**: the tiered build runs in the pinned container; the functional
  gates run against the tiered artifact; sizes are budgeted and
  journaled (engine wasm, core, academic).
- Provenance/reproducibility bars unchanged (MIT-only, pinned inputs;
  byte-repro remains descoped per ¬ß6.1 amendment).

## Risks / standing decisions

- ~~**`full` tier size + CI time is the marquee risk.**~~ **RESOLVED
  2026-07-24:** `full` deferred/not shipped for v1 (journal+CJK target
  doesn't need TL's long tail). Removes the multi-GB build-time/‚Äã
  footprint risk entirely. `academic` is the working set; the machinery
  stays N-tier-general for a later `full`. Remaining size watch is just
  `academic` itself (est. ~300‚Äì500 MB data pre-LZ4 ‚Äî latexextra +
  pictures dominate); it is on-demand, not preloaded, so it costs only
  when a doc needs it. Budget it at item 3.
- **Citation: `bibtex8`+`natbib` is the working framework** (not a gap).
  `bibtex8` is in the engine multicall; `natbib` is in
  `collection-latexrecommended` (‚Üí `academic`). This is a complete,
  journal-grade citing path ‚Äî `\citep`/`\citet`, author-year/numeric/‚Äã
  sorted styles, and every journal `.bst` (elsarticle, IEEEtran, revtex,
  achemso). Proven by the `bib-cite` corpus entry (engine‚Üíbibtex8‚Üí‚Ä¶).
  `biblatex` also ships (`collection-latexextra`) and works with
  `backend=bibtex` for many styles. **Made a first-class TESTED path in
  M4** (acceptance + corpus below).
  - **The one gap is `biber`** specifically: `biblatex` styles that
    REQUIRE the biber backend, and biber's advanced Unicode sort (a
    minor CJK-name-sorting wrinkle for the target ‚Äî many CN journals use
    citation-order/numeric styles where it doesn't bite). `biber` is a
    Perl program, not in the multicall; biber-to-WASM is a heavy
    STANDALONE effort, NOT scheduled in M4. Revisit as its own milestone
    only if target journals mandate biber.
- **`fandol` vs ¬ß6.3.** Bundling one CJK font (fandol) is a deliberate
  narrow exception to "fonts are host-supplied", justified by the CJK
  target (ctex must have a default font to compile). Recorded as a
  ¬ß6.3 refinement at item 3; production CJK fonts stay host-supplied.
- **On-demand post-init mount** (design note above) is the runtime
  technical unknown; item 5 spikes it first. If Emscripten can't mount
  a data package post-init in our setup, the fallback (WORKERFS or
  re-init) changes the ¬ß5.4 UX (a bundle load may cost a worker
  re-init) ‚Äî acceptable but must be journaled.
- **Disjointness vs kpathsea:** a file present in `full` but referenced
  by a `core` doc must resolve only after the tier loads ‚Äî the retry
  path (7) covers macro-driven refs the scan (6) misses. Tiers must not
  duplicate files (breaks the integrity manifest's one-file-one-hash
  assumption).
- **Reproducibility:** unchanged ‚Äî byte-repro stays descoped (¬ß6.1
  amendment); `SOURCE_DATE_EPOCH` + `normalize-lsr.py` keep the tiers
  mostly-deterministic for free. No local container builds.
- **Back-compat:** `texlive-basic.*` ‚Üí `core.*` rename keeps a
  generated alias for one release so the demo/0.0.1 consumers don't
  break; drop at M5.
