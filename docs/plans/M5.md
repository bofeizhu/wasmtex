# M5 — Release engineering + hardening: execution plan

Status: ACTIVE (opened 2026-07-24, after M4 closed). The FINAL milestone:
turn the accepted engine + runtime + bundle system into a real, verifiable,
documented **release** — versioned asset archives, an enumerated license
manifest for the shipped aggregate, size budgets, soak + browser-matrix
hardening, the fuller conformance corpus, and the docs an embedder needs —
then hand the actual tag / GitHub release / `npm publish` to the user
(those are user-only actions). DESIGN §9 M5 = "former M4 + M5"; PROMPT.md's
"M4 — Release engineering" + "M5 — Hardening" together are the concrete
checklist. Grounded in the 2026-07-24 readiness map (docs/LOG.md).

  export an `ASSETS_VERSION` constant from the package and have
  `createTypesetter` SOFT-verify the fetched `manifest.json` matches (clear
  error on mismatch, overridable). Decide at item 3/8.
- **[proposed defaults] Size-budget thresholds.** Current: busytex.wasm
  27.5 MB, core.data 53.9 MB, core.js 1.47 MB, academic.data 496 MB.
  Proposal: budget the PRELOAD path strictly (busytex.wasm ≤ 30 MB, core.data
  ≤ 60 MB, core.js ≤ 2 MB) and academic loosely (≤ 550 MB, on-demand). Tune
  at item 5.

## Acceptance (DESIGN §7/§8/§9 M5 + PROMPT M4/M5 checklists)

- **Versioned archives** produced reproducibly from a built `dist/`:
  `wasmtex-assets-<v>.tar.gz` (engine + formats + all bundles + manifest),
  and per-bundle `wasmtex-bundle-core-<v>.tar.gz` / `-academic-<v>.tar.gz`;
  each archive's contents verify against `manifest.json`.
- **License audit of the shipped aggregate**: THIRD_PARTY_NOTICES.md
  enumerates the license of every TeX Live package actually in core+academic
  (from tlpdb `catalogue-license`); a CI check fails if a shipped package has
  no recorded license (or a disallowed one). The §7 aggregate-distribution
  statement stands.
- **Docs**: root README rewritten (accurate status), a real embedding guide
  (§10 profile — assetsBaseUrl, custom scheme, cold-start, integrity
  verification, no-network), runtime/README asset-channel language flipped.
- **Size budgets**: a checked-in budget + a CI check diffing manifest `bytes`
  against it; growth of the preload path fails loud.
- **Hardening**: soak test (50 sequential jobs w/ random cancellations — no
  cross-job contamination, memory returns to baseline after `dispose()`);
  browser matrix (chromium + firefox + webkit) incl. a REAL-browser on-demand
  academic mount; the fuller conformance corpus (unicode-math, multi-chapter
  `\include`, a known-bad doc, a standalone TikZ figure, a host-supplied-font
  CJK variant).
- **Release dry-run green**: `npm pack`/`--dry-run` produces the expected
  tarball; the release workflow builds + attaches archives on a tag — proven
  by a dry run / dispatch, WITHOUT publishing.
- **The release itself** (tag `assets-v<v>`, GitHub Release, `npm publish`)
  is executed by the USER; M5 is marked complete when the prep is green and
  the release is cut.

## Risks / standing decisions

- **User-only actions gate the finish.** Tag, GitHub Release, and
  `npm publish` are the user's calls (standing rule since M2). M5 does ALL
  prep + a dry run; the release moment is a handoff, not autonomous.
- **License enumeration correctness is load-bearing** (it's a legal
  statement about what we ship). `catalogue-license` can be missing/`noinfo`
  for some packages — the audit must FAIL on those, not guess, so a human
  resolves each before release. This may surface packages we must document
  or drop from `academic`.
- **No local container builds** (standing directive) — release-archive
  packing runs on a built dist (native for dev, container for the real
  release via item 8's workflow). The versioned archives that ship come from
  the pinned container only (DESIGN §9 floor).
- **texlive-basic alias drop** is a breaking change for any 0.0.1 consumer
  that used it — acceptable pre-1.0 (name-reservation release), noted in the
  release notes; the demo migrates to core+academic in the same change.
- **Reproducibility stays descoped** (§6.1 amendment); archives are packed
  deterministically (sorted, SOURCE_DATE_EPOCH) but not gated byte-identical.
